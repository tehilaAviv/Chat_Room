Explanation of the exercise:

In the exercise we implemented 2 pages for the user: a login page and a chat room page - where he can send messages, see the connected users and search for messages by name and content.
Classes we created:
# Login class - manages the session and is of the bean type. We used it in the other departments to check if a user is logged in or not.
# ControllerRoute - A department responsible for user management, user login and logout.
# DbController class - handles database management, data entry and retrieval of data from the message table and the user table using various queries.
# Error class - is responsible for redirecting the user to an error page in case of an error or if he accesses an incorrect address.
# Message Class - Defines the fields in the database message table as well as the field update functions.
# MessageRepository - Defines the various queries we used to retrieve data from the message table.
# Users class - defines the fields in the database of users as well as the functions for updating the fields.
# UsersRepository class - Defines the various queries we used to retrieve data from the user table.
# Filter class - Used to move the user to a login page in case the user wants to move to another page that is not a login page and he has not logged in before. The handling in this ward is done through the MyConfig ward.
# We also implemented in the file: display.js the functions that json returns to us and are used to print the sent messages, the messages found in the searches and to print the connected usersIn the exercise we implemented 2 pages for the user: a login page and a chat room page - where he can send messages, see the connected users and search for messages by name and content.
